XinFin XDPoS relies on a system of 108 Masternodes with XDPoS consensus that can support low transaction fees and 2-second transaction confirmation times. At current setup the number of masternode has been limited to 108. Currently there are already 108 masternodes which power XinFin XDPoS.

Is there any option to increase the number of MasterNode beyond 108?

No, currently, the number of XinFin Masternodes are limited strictly to 108 Masternodes.

What is the Purpose of Stand-by Node or Backup?

Standby Node or Backup node make the Blockchain Network more robust and stable. If any Masternode / Validators nodes resign or go out of the network then Standby Node becomes a masternode and starts participating to write the Network Transaction. This feature makes the Network more robust and stable.

What is the current limit set for Stand-by MasterNode?

Currently, The number of Standby Masternode are limited strictly to 250. So you have a very limited time period to order nodes and setup it up.
